# HMH Into Math TX - Complete Vendor Checklist
**Source:** Into Math TX Vendor Checklist v2.docx
**Purpose:** Quality assurance validation criteria for all lesson content blocks

---

## 1. Learning Goals

**Purpose:** To help students self-assess confidence and to clarify lesson objectives.

### Student Edition Requirements:
- ✅ Includes **I Can statement** (student-friendly language)
- ✅ Bold key math vocabulary
- ✅ Provides rating scale with 3 options:
  - "I don't understand yet"
  - "I need more practice"
  - "I've got it"

### Teacher Edition Requirements:
- ✅ Begins with **START** boilerplate for Learning Goal routine
- ✅ Clarifies 1-2 new vocabulary terms using glossary definitions
- ✅ Includes 3-5 steps minimum:
  1. Read Learning Goal aloud
  2. Clarify key terms
  3. Students self-assess using rating scale
  4. Review results
  5. Revisit at end of lesson
- ✅ Ends with instruction to revisit Learning Goal

### Example:
**SE:** "I can compose two-dimensional shapes with given properties."
**TE:** "Clarify 'compose' by explaining it means to put together shapes to form a new one."

## 2. Before You Teach

**Purpose:** To provide teachers with conceptual depth, prerequisite knowledge, objectives, standards, and supports.

### Teacher Edition Only - Requirements:

**"What should I understand about the math?"**
- ✅ ≤420 characters total
- ✅ Exactly 3 bullets
- ✅ Conceptual depth about the mathematics
- Example: "Emphasize attributes such as sides and angles when composing shapes."

**Prior Learning**
- ✅ Exactly 3 bullets
- ✅ Past tense verbs ("Students learned...", "Students explored...")
- ✅ Lowercase starts (not capitalized)

**Standards & Objectives**
- ✅ Focus Standard(s) stated verbatim from TEKS breakout
- ✅ ELPS code + full text included
- ✅ Learning Objective provided
- ✅ Language Objective provided

**Future Connections**
- ✅ Exactly 3 bullets
- ✅ "will..." phrasing ("Students will...", "This will...")

**Materials**
- ✅ Manipulatives list organized by:
  - Per student
  - Per group
  - Digital

## 3. Spark Your Learning (SYL)

**Purpose:** To introduce a lesson concept using prior knowledge and problem-solving routines.

### Student Edition Requirements:

**Problem**
- ✅ ≤120 words maximum
- ✅ Solvable with prerequisite knowledge only
- ✅ Bold math vocabulary
- ✅ Ends with "Show your thinking."

**Visual Assets**
- ✅ Spec Type specified (art, photo, or tech art)
- ✅ Spec Description included
- ✅ Alt Text provided if visual used

**Turn & Talk**
- ✅ Included with annotation

### Teacher Edition Requirements:

**Routine Start**
- ✅ Begins with **START** boilerplate
- ✅ **MLR6: Three Reads** routine included:
  - Read 1: Context
  - Read 2: Quantities
  - Read 3: Question

**Spark Discussions - Common Error**
- ✅ Student error example shown
- ✅ Teacher Guide script provided
- ✅ 3 ASK questions minimum
- ✅ Spec Type, Description, Alt Text if visual needed

**Spark Discussions - Deepening Student Thinking**
- ✅ Correct student example shown
- ✅ Teacher Guide script provided
- ✅ 3 ASK questions minimum
- ✅ Spec Type, Description, Alt Text if visual needed

**Turn & Talk**
- ✅ Teacher Guide included

**Multilingual Learner Scaffolds**
- ✅ 3 proficiency levels provided:
  - Emerging
  - Expanding
  - Bridging
- ✅ Scaffolded examples for each level

**Additional Supports**
- ✅ 1 UDL strategy included
- ✅ Vocabulary clarification provided

### Example:
**SE:** "Maria has 2 triangles and 1 square. How can she make a house shape?"
**TE ASK:** "What numbers are important? Possible answer: 2 triangles, 1 square."

## 4. Review Spark Your Learning

**Purpose:** To revisit the Spark Your Learning problem and guide reflection.

### Student Edition Requirements:
- ✅ Restates Spark Your Learning problem (shortened form acceptable)
- ✅ Adds reflection question (e.g., "How would you check your work?")
- ✅ Spec Type, Description, Alt Text if visual used

### Teacher Edition Requirements:
- ✅ Begins with **NOW, review Spark Your Learning** boilerplate
- ✅ Includes at least 3 ASK questions about:
  - Checking work
  - Operations used
  - Solution verification
- ✅ Closes with feedback guidance
- ✅ Notes connection to Differentiation & Practice
- ✅ Multilingual Learner strategy integrated:
  - Peer/group sharing
  - Respectful conversation prompts

### Example:
**TE ASK:** "What operation would you use to check division? Possible answer: Multiplication."

---

## 5. Quick Check

**Purpose:** To formatively assess mastery of the lesson's Learning Goal and Language Objective.

### Student Edition Requirements:
- ✅ Interactions numbered sequentially (Interaction 1, Interaction 2, etc.)
- ✅ Each Interaction includes:
  - SE Manuscript
  - Annotation
- ✅ Spec Type + Description if visual used
- ✅ Bold math vocabulary

### Teacher Edition Requirements:
- ✅ Begins with **NOW, use the Quick Check** boilerplate
- ✅ Language Objective reference/hook included
- ✅ Teaching Strategy: Multilingual Learners
  - Tied to specific ELPS
  - Includes sentence frames/starters/word banks
- ✅ Optional: UDL or Common Error support (if provided in data inputs)

### Example:
**SE:** "There are 46 swimmers and 7 lanes. How many swimmers per lane?"
**TE:** "NOW, use the Quick Check... Ask students to orally explain repeated subtraction."

## 6. Tasks

**Purpose:** To provide scaffolded practice aligned to TEKS, progressing in rigor, with SE/TE parity.

### Student Edition Requirements:

**Problem Stem**
- ✅ ≤150 words maximum
- ✅ Bold math vocabulary

**Sub-Items**
- ✅ Labeled A, B, C, etc.
- ✅ Each includes annotation
- ✅ Spec Type, Description, Alt Text if visual used

**Turn & Talk**
- ✅ Included with annotation

### Teacher Edition Requirements:

**Routine Start**
- ✅ Begins with **START** boilerplate
- ✅ **MLR6: Three Reads** routine OR alternative MLR

**Teacher Guide Questions**
- ✅ Match each SE sub-item (A, B, C)
- ✅ ≤5 questions per sub-item
- ✅ Include "Possible answer:" for each

**Process Standards**
- ✅ **NEXT:** Texas Process Standard citation (verbatim from TEKS)
- ✅ Support notes for implementing the process standard

**Problem Solving**
- ✅ Ends with **NOW, have students solve the problem** boilerplate

**Additional Supports**
- ✅ Turn & Talk Teacher Guide
- ✅ MLR routine (aligned to last item in task)
- ✅ ML/UDL/Common Error support (with item alignment specified)

**DOK Leveling (Task 1 ONLY)**
- ✅ DOK Level 1: Recall
- ✅ DOK Level 2: Basic Application of Skills & Concepts
- ✅ DOK Level 3: Strategic Thinking and Complex Reasoning
- ✅ Each level includes annotation

### Example:
**SE:** "Lisa has $50 to buy goggles that cost $8 each. How many can she buy?"
**TE ASK:** "What numbers are important? Possible answer: 50 and 8."

---

## 7. Practice On Your Own (POYO)

**Purpose:** To give students independent practice and provide TE with Item Guides and Rubrics.

### Student Edition Requirements:
- ✅ Interactions labeled (Interaction 1, Interaction 2, etc.)
- ✅ Each Interaction includes:
  - SE Manuscript
  - Annotation
- ✅ Spec Type + Description if visual used

### Teacher Edition Requirements:

**Item Guide**
- ✅ Table with columns:
  - Item # (e.g., "3a")
  - DOK Level (1, 2, or 3)
  - Task Alignment (which task this connects to)

**Rubrics**
- ✅ At least one rubric included
- ✅ 2/1/0 point scale:
  - 2 points: Complete/correct with full explanation
  - 1 point: Partial or minor errors
  - 0 points: Incorrect or incomplete
- ✅ Sample answers provided for each point level
- ✅ **At least one rubric aligned to a modeling or visual item**

### Example:
**SE:** "Use repeated subtraction to solve 46 ÷ 5. Possible answer: 9 r 1."
**TE Item Guide:** "3a — DOK 2 — Task 2"
**TE Rubric:** "2 pts for complete number line model, 1 pt partial, 0 pts incorrect."

## Validation Checklist Summary

Use this quick reference to validate generated content:

### Character/Word Limits
- [ ] SYL Problem: ≤120 words
- [ ] Task Problem: ≤150 words
- [ ] "What should I understand": ≤420 characters
- [ ] Prior Learning: 3 bullets
- [ ] Future Connections: 3 bullets

### Required Boilerplate
- [ ] **START** used for: Learning Goals, SYL, Tasks
- [ ] **NOW** used for: Review SYL, Quick Check
- [ ] **NEXT** used for: Process Standards in Tasks

### Structural Elements
- [ ] MLR6 Three Reads in: SYL, Tasks
- [ ] Turn & Talk in: SYL, Tasks (with both SE and TE)
- [ ] EB Scaffolds: 3 levels (Emerging, Expanding, Bridging) in SYL
- [ ] ASK Questions: ≥3 per section
- [ ] TG Questions: ≤5 per sub-item
- [ ] UDL: 1 strategy in SYL
- [ ] DOK Levels 1, 2, 3: Task 1 only
- [ ] Item Guide: All POYO items
- [ ] Rubric: At least 1 with 2/1/0 scale

### Visual Assets
- [ ] All visuals have: Spec Type + Description + Alt Text
- [ ] All SE content has: Annotations

### Vocabulary
- [ ] Bold new vocabulary throughout
- [ ] Vocabulary clarified in TE Learning Goals
- [ ] Vocabulary clarified in TE SYL

## Common Validation Errors

**Character Limit Violations:**
- SYL problems exceeding 120 words
- Task problems exceeding 150 words
- "What should I understand" exceeding 420 characters

**Missing Boilerplate:**
- START boilerplate not at beginning of sections
- NOW boilerplate missing from Review SYL
- NEXT boilerplate missing before Process Standards

**Incomplete Scaffolds:**
- EB scaffolds missing one of three proficiency levels
- Fewer than 3 ASK questions in a section
- DOK leveling missing from Task 1

**Visual Asset Issues:**
- Missing Spec Type, Description, or Alt Text
- Missing annotations on SE content

**Structural Violations:**
- Turn & Talk missing TE guide
- MLR6 Three Reads not included in SYL
- Item Guide missing from POYO

---

**Validation Priority:**
1. Character/word limits (hard limits)
2. Required boilerplate (must be exact)
3. EB scaffolds (must have all 3 levels)
4. DOK leveling (Task 1 only)
5. Visual assets (complete spec + alt text)
